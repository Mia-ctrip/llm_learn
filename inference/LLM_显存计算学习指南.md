# LLM 显存计算学习指南

## 1. 模型大小定义

### 1.1 参数量（Parameters）

**B = Billion = 十亿**

- **7B** = 70亿参数
- **13B** = 130亿参数  
- **14B** = 140亿参数
- **27B** = 270亿参数
- **35B** = 350亿参数
- **72B** = 720亿参数

### 1.2 如何理解参数量

模型的参数就像神经网络中的"权重"，每个参数都需要存储和计算。

参数量 = 所有权重矩阵中元素的总数

常见的 Transformer 模型参数量估算：
- **Embedding层**：vocab_size × hidden_dim
- **Attention层**：每层大约 hidden_dim²
- **MLP层**：每层大约 hidden_dim × 4 × hidden_dim
- 总参数 ≈ 12 × hidden_dim × num_layers

**例子**：
- Qwen2.5-7B 有 **70亿个参数**
- Qwen2.5-14B 有 **140亿个参数**（大约是7B的2倍）

---

## 2. 模型精度（数据类型）
模型的精度就是模型权重的精度

### 2.1 常见精度类型

| 精度 | 字节数 | 能表示的范围 | 备注 |
|------|--------|-----------|------|
| **FP32** (float32) | 4 字节 | ±3.4×10³⁸ | 全精度，精度最高，显存最多 |
| **FP16** (float16) | 2 字节 | ±65504 | 半精度，可能精度丧失 |
| **BF16** (bfloat16) | 2 字节 | ±3.4×10³⁸ | 谷歌提出，兼具FP32范围和FP16效率 |
| **INT8** (int8) | 1 字节 | -128 ~ 127 | 量化，显存最少，推理速度快 |
| **FP8** (float8) | 1 字节 | ±240 | 超轻量化 |

### 2.2 精度对显存的影响

**模型权重占用的显存 = 参数量 × 精度字节数**

例子：以Qwen2.5-7B为例（70亿参数）

- **FP32 精度**：7B × 4 字节 = **28GB**
- **FP16 精度**：7B × 2 字节 = **14GB**
- **INT8 精度**：7B × 1 字节 = **7GB**
- **FP8 精度**：7B × 1 字节 = **7GB**

### 2.3 精度选择建议

- **训练**：通常用 FP32 或混合精度 (FP16 + FP32)
- **推理**：可以用 FP16、BF16、INT8、FP8
- **移动端**：INT8、FP8

---

## 3. 推理时的显存组成

推理时总显存 = **模型权重 + KV Cache + 运行时显存**

### 3.1 模型权重（Model Weights）

**公式**：权重显存 = 参数量 × 精度字节数

这是**固定的**，和输入长度、batch size 无关。

**例子**：
- Qwen2.5-7B (FP16) = 7B × 2 = 14GB
- Qwen2.5-14B (FP16) = 14B × 2 = 28GB

### 3.2 KV Cache（关键值缓存）

#### 什么是 KV Cache？

在 Transformer 中，Attention 机制需要计算当前 token 与所有历史 token 的相似度。
- **K** = Key （历史 token 的查询键）
- **V** = Value （历史 token 的特征向量）

每生成一个 token，都要保存该 token 的 K 和 V，这就是 KV Cache。

#### KV Cache 显存计算

**公式**：
```
KV Cache 显存 = 2 × (batch_size) × (sequence_length) × (hidden_dim) × (num_layers) × (精度字节数)
              = 2 × batch × seq_len × hidden_dim × num_layers × bytes_per_token

简化公式：
KV Cache 显存 ≈ 2 × 参数量 × (seq_len / hidden_size)

实用公式：
KV Cache 显存 ≈ batch_size × seq_len × 参数量 × 精度字节数 / (1000M × hidden_size系数)
```

#### 实际计算示例

**Qwen2.5-7B**
- 参数量：7B
- hidden_size：3584
- num_layers：28
- 精度：FP16 (2字节)

**单个 token 的 KV Cache**：
```
= 2 × 1(batch) × 1(token) × 3584(hidden) × 28(layers) × 2(bytes)
= 2 × 3584 × 28 × 2
≈ 0.4 MB
```

**1000个 tokens 的 KV Cache**：
```
= 2 × 1 × 1000 × 3584 × 28 × 2
≈ 403 MB
```

**2K tokens (2000) 的 KV Cache**：
```
= 2 × 1 × 2000 × 3584 × 28 × 2
≈ 806 MB ≈ 0.8GB
```

**4K tokens (4000) 的 KV Cache**：
```
= 2 × 1 × 4000 × 3584 × 28 × 2
≈ 1.6 GB
```

**Batch=4, 2K tokens 的 KV Cache**：
```
= 2 × 4 × 2000 × 3584 × 28 × 2
≈ 3.2 GB
```

#### 快速记忆法

对于一般的大模型（参数量为 P，以 B 计），**单个 batch 的 KV Cache**：

```
KV Cache (GB) ≈ seq_len × batch_size × 2 / 1000

例如：
- 1个batch，2K tokens，7B模型 → KV Cache ≈ 0.8 GB
- 4个batch，2K tokens，7B模型 → KV Cache ≈ 3.2 GB
```

### 3.3 运行时显存（Runtime Memory）

临时显存，用于计算中间结果，包括：
- 注意力权重计算
- MLP 激活值
- 梯度（仅训练）

**估算**：运行时显存 ≈ 模型权重的 10-20%

**例子**：
- Qwen2.5-7B (FP16) 权重 = 14GB
- 运行时显存 ≈ 1.4GB - 2.8GB

## 4. 推理显存总计公式

```
总显存 (GB) = 权重显存 + KV_Cache + 运行时显存

详细版本：
总显存 = (参数量 × 精度字节数) 
       + (2 × batch × seq_len × hidden_dim × layers × 精度字节数 / 1B)
       + (权重显存 × 10-20%)
```

### 4.1 实例计算：Qwen2.5-7B FP16

**配置**：
- 模型：Qwen2.5-7B
- 精度：FP16（2字节）
- Batch size：1
- Sequence length：2000 tokens
- hidden_dim：3584
- num_layers：28

**计算**：

1. **权重**：7B × 2 = **14 GB**

2. **KV Cache**：
   ```
   = 2 × 1 × 2000 × 3584 × 28 × 2 / (1000M)
   ≈ 0.8 GB
   ```

3. **运行时**：14 × 0.15 = **2.1 GB**

4. **总计**：14 + 0.8 + 2.1 = **16.9 GB** → **≈ 17 GB**

### 4.2 实例计算：Qwen2.5-14B FP16

**配置**：同上，但模型是14B

1. **权重**：14B × 2 = **28 GB**

2. **KV Cache**：
   ```
   = 2 × 1 × 2000 × 3584 × 28 × 2 / (1000M)
   ≈ 0.8 GB  （这个不变，因为hidden_dim相同）
   ```

3. **运行时**：28 × 0.15 = **4.2 GB**

4. **总计**：28 + 0.8 + 4.2 = **33 GB** → 需要 **2 × 16GB显卡** 或 **1 × 40GB显卡**

| **T4** | 16GB | 7B模型推理（batch=1）|

## 6. 训练时的显存需求

训练显存 = 权重 + 梯度 + 优化器状态 + KV Cache + 中间激活值 + 加载数据

**典型的训练显存 ≈ 模型权重 × 4-6倍**

例子：
- Qwen2.5-7B FP16 权重 = 14GB
- 训练需要 ≈ 56-84GB （通常需要多张卡）

## 8. 实战步骤：给定显卡，快速确定能跑的模型

### 情景：你有一张 24GB L20 显卡，想选模型

**步骤**：

1. **预留系统开销**：24GB × 80% = **19.2GB 可用**

2. **假设常见配置**：
   - 精度：FP16
   - Batch size：1
   - Seq_len：2000 tokens

3. **逐个检验**：
   - 7B：14 + 0.8 + 2 = 16.8GB ✓ **可以**
   - 13B：26 + 0.8 + 4 = 30.8GB ✗ **超了**

4. **结论**：L20(24GB) 适合 **7B 模型**

### 情景：你想跑 Qwen2.5-14B，需要什么显卡？

**步骤**：

1. **计算显存需求**（FP16, Batch=1）：
   - 权重：28GB
   - KV Cache：0.8GB
   - 运行时：4.2GB
   - **总计：33GB**

2. **选择显卡**：
   - 单卡方案：需要 **≥40GB** → A100 或 H20
   - 双卡方案：2×T4(32GB) **刚好** 或 2×L20(48GB) **充足**

3. **推荐**：选择 A100_1_40 或 H20_1_96（更富裕）

## 9. 量化的影响

如果使用 **INT8 量化**，显存减半：

| 模型 | FP16 | INT8 | 推荐显卡 |
|------|------|------|--------|
| 7B | 17GB | **8.5GB** | T4(16GB) |
| 14B | 33GB | **16.5GB** | T4(16GB) × 2 或 L20(24GB) |
| 72B | 167GB | **83GB** | H20(96GB) × 1 |

**量化的代价**：推理精度略降，但速度可能提升。

---

## 10. 总结：记住这三个公式

### 公式 1：模型权重
```
权重显存 (GB) = 参数量(B) × 精度字节数
```

### 公式 2：KV Cache（快速版）
```
KV Cache (GB) ≈ Batch_size × Seq_len × 2 / 1000
```

### 公式 3：推理总显存
```
总显存 (GB) = 权重 + KV Cache + (权重 × 15%)
```
