## 四、你的部署策略解读

### 策略 1：L20_1_48 的统治地位

**为什么 60% 的服务都用 L20_1_48？**

推测：
1. **价格/性能比最优**（相比 A100、H20）
2. **可以承载 7B-14B 模型**（常见规模）

- 如果你也在部署，L20_1_48 应该是首选

- 大模型（70B）用 H20
- 特定高精度任务用 A100

推测：
- 不同的业务线有不同的 SLA
- 关键路径使用更好的硬件

## 五、量化的证据

### 为什么 Qwen3(70B) 能在 H20_1_96 上跑？

理论上 Qwen3 70B FP16 需要 160GB，但你的数据中有：

```
Qwen3 + H20_1_96（96GB）
```

**只能是**：
1. **INT8 量化**：70B × 1 byte = 70GB ✓
2. **INT4 量化**：70B × 0.5 byte = 35GB ✓
3. **GPTQ 量化**：如 `Qwen/Qwen3.5-35B-A3B-GPTQ-Int4`（你数据中出现过）
4. **Flash Attention** 减少 KV Cache

**这说明**：你的生产环境中大量使用了量化技术！
