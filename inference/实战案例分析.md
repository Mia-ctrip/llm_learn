# 从真实部署数据分析

## 从 model_usage.guide.txt 提取的实例分析

### 案例 1：Qwen2.5-14B + L20_1_48

**数据行**：Qwen2.5-14B 部署在 L20_1_48

**分析**：

1. **模型配置**：
   - 模型：Qwen2.5-14B
   - 参数量：140 亿
   - 通常精度：FP16（业界标准推理精度）

2. **显存估算**（假设推理场景，Batch=1, Seq=2K）：
   - 权重：14B × 2 bytes = **28GB**
   - KV Cache：≈ **0.8GB**
   - 运行时：28 × 15% = **4.2GB**
   - **总计：33GB**

3. **显卡配置**：L20_1_48 = 1 × 48GB
   - 可用显存：48GB
   - 占用率：33/48 = **68.75%** ✓ 良好
   - 富余度：15GB（足以支持更长序列或更大batch）

4. **实际意义**：
   - ✓ 可以支持更长的输入序列（4K, 8K tokens）
   - ✓ 可以支持 batch_size=2-3
   - ✓ 有空间用于系统缓存

**结论**：L20_1_48 是 Qwen2.5-14B 的 **最佳搭配**

### 案例 2：Qwen3 + H20_1_96

**数据行**：Qwen3 部署在 H20_1_96（多次出现，包括 H20_8_768）

**分析**：

1. **模型配置**：
   - 模型：Qwen3（通常指 Qwen3 70B 版本）
   - 参数量：700 亿（通常）
   - 精度：FP16

2. **显存估算**（推理，Batch=1, Seq=2K）：
   - 权重：70B × 2 bytes = **140GB**
   - KV Cache：≈ **0.8GB**
   - 运行时：140 × 15% = **21GB**
   - **总计：≈162GB**

3. **显卡配置**：
   - 单卡 H20_1_96 = 96GB → **不足** ✗
   - 双卡 H20_2_192 = 192GB → **足够** ✓
   - 8卡 H20_8_768 = 768GB → **绰绰有余** ✓

4. **你数据中的情况**：
   - H20_1_96（1卡）：可能是 **量化版本**（INT8）
     - 权重：70B × 1 = 70GB
     - KV + Runtime：≈ 25GB
     - 总计：≈95GB（勉强）
   - H20_8_768（8卡）：**完整FP16部署**，用于大batch或推理优化

**结论**：
- Qwen3 70B 单卡 FP16 需要 **至少 192GB**（需要 2 张 H20）
- 如果用 1 张 H20_1_96，**必须量化到 INT8** 或使用 Flash Attention

### 案例 3：Qwen2.5-7B + T4_1_16

**数据行**：多个小模型 + T4_1_16

**分析**：

1. **模型配置**：
   - 模型：Qwen2.5-7B
   - 参数量：70 亿
   - 精度：FP16

2. **显存估算**（推理，Batch=1, Seq=2K）：
   - 权重：7B × 2 = **14GB**
   - KV Cache：≈ **0.8GB**
   - 运行时：14 × 15% = **2.1GB**
   - **总计：16.9GB**

3. **显卡配置**：T4_1_16 = 16GB
   - 占用率：16.9/16 = **105.6%** ⚠️ **刚好溢出**

4. **实际可行性**：
   - 如果显存管理严格：基本能跑，但**没有余地**
   - 如果 Seq_len 更长（4K）：会 OOM
   - 如果 Batch_size > 1：会 OOM
   - 实际可能用的是 **INT8 量化** → 7B × 1 = 7GB（充足）
   - 或者 **Flash Attention 优化**，减少 KV Cache 峰值

**结论**：T4(16GB) + 7B 很紧张，应该：
- ✓ 用 INT8 量化
- ✓ 启用 Flash Attention
- ✗ 不要长序列
- ✗ 不要大 batch

### 案例 4：bge-large（300M）+ L20_1_48

**数据行**：Embedding 模型 bge-large（300M）+ L20_1_48

**分析**：

1. **模型配置**：
   - 模型：bge-large（Embedding 模型）
   - 参数量：3 亿（300M = 0.3B）
   - 精度：FP32（Embedding 模型通常用FP32）

2. **显存估算**（推理，Batch=32, Seq=512）：
   - 权重：0.3B × 4 = **1.2GB**
   - KV Cache：Embedding 模型无此概念（只有encoder，无decoder）
   - 运行时（Batch处理）：1.2 × 2 = **2.4GB**
   - **总计：3.6GB**

3. **显卡配置**：L20_1_48 = 48GB
   - 占用率：3.6/48 = **7.5%** ✓ 绰绰有余
   - 可以支持 Batch=256+ 或多个并发请求

**结论**：Embedding 模型显存压力很小，L20_1_48 严重过度配置，通常会和其他模型混合部署

## 实战习题

### 习题 1：你现在有 L20_1_48，想部署什么最大的模型？

**思路**：

1. 预留 20% 系统开销：48GB × 80% = 38.4GB
2. 假设常见配置（FP16, Batch=1, Seq=2K）
3. 总显存公式 = 权重 × 2 + 0.8 + 权重 × 2 × 0.15

求解：
```
参数量 × 2 × 1.15 + 0.8 ≤ 38.4
参数量 × 2.3 ≤ 37.6
参数量 ≤ 16.3B
```

**答案**：可以部署 **Qwen2.5-14B** ✓ 或 **Mistral-7B** ✓ 或 **Llama3-8B** ✓

**不能部署**：Qwen2.5-27B（会溅出）

---

### 习题 2：Qwen2.5-72B 用什么显卡配置？

**思路**：

1. FP16 推理（Batch=1, Seq=2K）
   - 权重：72B × 2 = 144GB
   - 其他：≈ 22GB
   - **总计：166GB**

2. 选择方案：
   - ✓ 双 H20（2×96GB=192GB）
   - ✓ 双 A100 PCIe（2×40GB=80GB 不足）→ 需要 4 张
   - ✓ INT8 量化 + 单 H20（72×1 + 25 ≈ 97GB）
   - ✗ 单张显卡无法承载

**答案**：**双 H20（H20_2_192）** 或 **INT8 + 单 H20**

---

### 习题 3：如果要支持 Batch_size=4，Qwen2.5-14B 需要什么显卡？

**思路**：

1. FP16 推理（Batch=4, Seq=2K）
   - 权重：28GB（不变）
   - KV Cache：2 × 4 × 2000 / 1000 ≈ **3.2GB**
   - 运行时：4.2GB（略增）
   - **总计：≈35.4GB**

2. 选择方案：
   - ✓ A100（40GB）
   - ✓ L20_1_48（48GB）
   - ✗ 2×T4（32GB，不足）

**答案**：**需要升级到 A100 或 H20 或 L20_1_48**

## 从真实数据推导的规律

### 规律 1：同一模型的不同显卡配置

观察 Qwen2.5-14B：
- L20_1_48：部署1份
- A100_1_80：部署多份
- L20_2_96：部署多份

**解释**：
- L20_1_48 是最小化单位（勉强足够）
- A100 和 L20_2_96 提供了冗余（可能支持更大batch或更长序列）

### 规律 2：Embedding 模型的特点

你数据中：
```
bge-large×4 + L20_1_48
bge-m3 + T4_1_16
```

**解释**：
- Embedding 模型显存占用极低（通常 <2GB）
- 通常和其他模型混合部署
- 显卡选择主要看其他模型

### 规律 3：大模型部署的倍数关系

观察数据：
- H20_1_96（1卡）
- H20_2_192（2卡）
- H20_8_768（8卡）

**解释**：
- 大型推理集群会用多卡并行
- 卡数 = ceil(总显存需求 / 单卡容量)
