### 第 2-3 分钟：理解核心公式

记住这三个公式（就这样做，直接背）：

```
公式 1：权重显存 = 参数量(B) × 精度字节数
       例：7B × 2 = 14GB（FP16）

公式 2：KV Cache ≈ Batch × Seq_len / 1000 (GB)
       例：1 × 2000 / 1000 = 2GB

公式 3：推理总显存 ≈ 权重 × 2.5（包括KV和运行时）
       例：7B × 2.5 = 17.5GB
```

### 第 4-5 分钟：快速验证

用这个速记：

```
7B FP16  → ~17GB  → T4(16GB) 或更好
14B FP16 → ~33GB  → L20(48GB) 或 A100
70B FP16 → ~160GB → 2×H20 或 INT8 单卡
```

**就这样！你已经掌握了 80% 的知识。**

## 💡 核心概念速记

### 概念 1：参数量（B）= Billion（十亿）

```
7B = 70 亿参数
14B = 140 亿参数
越大 → 模型越强 → 但显存越多
```

### 概念 2：精度 = 显存乘数

```
FP32：1 参数 = 4 字节
FP16：1 参数 = 2 字节  ← 推理标准
INT8：1 参数 = 1 字节  ← 节省 50%
INT4：1 参数 = 0.5 字节 ← 节省 75%
```

### 概念 3：显存三部分

```
权重      = 固定的，参数 × 精度
KV Cache = 动态的，随 batch 和序列增长
运行时    = 约权重的 15%
```
