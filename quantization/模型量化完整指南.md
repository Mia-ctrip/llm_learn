# 模型量化完整指南

## 1. 什么是模型量化？

### 定义

模型量化是将训练好的深度学习模型的参数从高精度浮点数（如 float32）转换为低精度数字（如 int8、int4）的技术。

**不是简单的四舍五入**，而是**科学的线性映射**。

### 核心目标

| 目标 | 效果 |
|------|------|
| 减少模型显存占用 | float32 → int8 可节省 75% 显存 |
| 加速推理速度 | 整数计算比浮点数计算快 2-3 倍 |
| 支持更小的硬件 | 在消费级 GPU 或 CPU 上运行大模型 |
| 保持精度 | 精度损失通常 <2% |

### 显存节省示例

```
模型: 13B 参数 LLM

float32:  13B × 4 字节 = 52 GB
float16:  13B × 2 字节 = 26 GB  (50% 节省)
int8:     13B × 1 字节 = 13 GB  (75% 节省)
int4:     13B × 0.5 字节 = 6.5 GB (87.5% 节省)
```

## 2. 量化的理论基础

### 2.1 量化的数学本质

量化 = **将浮点数范围线性映射到整数范围**

#### 非对称量化公式（最常用）

```
第一步：确定权重范围
min_val = min(weights)
max_val = max(weights)
range = max_val - min_val

第二步：计算映射参数
scale = range / (2^bits - 1)        # bits 通常为 8（int8）或 4（int4）
zero_point = -min_val / scale

第三步：量化
int_value = round((float_value - min_val) / scale + zero_point)

第四步：推理时反量化
recovered_float = (int_value - zero_point) * scale
```

#### 数学推导

```
原公式：
int_value = round((float_value - min_val) / scale + zero_point)

代入 zero_point = -min_val / scale：
= round((float_value - min_val) / scale - min_val / scale)
= round((float_value - 2*min_val) / scale)
= round(float_value/scale + 2*(-min_val/scale))
= round(float_value/scale + 2*zero_point)
```

### 2.2 为什么用实际权重范围而不是浮点类型范围？

#### 你的方法（基于 float32 理论范围）

```
float32 范围：[-3.4e38, 3.4e38]

权重 0.15 的位置 = (0.15 - (-3.4e38)) / (6.8e38)
                ≈ 50% (非常接近)

int8 值都堆积在 50% 附近
结果：256 个 int8 值中只用了 3 个，浪费 98% ❌
精度：极低，不可用
```

#### 实际方法（基于权重实际分布）

```
权重实际范围：[-0.5, 0.5]

权重 0.15 的位置 = (0.15 - (-0.5)) / (0.5 - (-0.5))
                = 0.65 / 1.0
                = 65%

int8 值均匀分布在 [0, 255]
结果：充分利用所有 256 个值 ✓
精度：高，精度损失 <2%
```

#### 原因分析

```
1. 精度需求
   ─────────
   你的方法：每个 int8 值代表 2.7e36 的 float 范围
   实际方法：每个 int8 值代表 0.004 的 float 范围
   分辨率差异：100 亿倍！

2. 神经网络的真实特性
   ────────────────
   权重初始化（Xavier/He）：
   std = sqrt(2 / (fan_in + fan_out))
   
   示例：1000→1000 层
   std ≈ 0.032
   99% 的权重在 [-0.1, 0.1] 范围内
   
   权重从不会均匀分布在 float32 全体范围内
```

### 2.3 zero_point 的几何意义

zero_point 是一个**坐标轴平移参数**。

#### 数轴表示

```
对称量化（无 zero_point）：
float:        [-2  -1   0   1   2   3]
              ↓   ↓   ↓   ↓   ↓   ↓
缩放后：    [-102 -51  0  51 102 153]
int8:   [-128,-102,-51, 0, 51,102,127]
                ↑              ↑
            跨越负数，浪费空间

非对称量化（有 zero_point=102）：
float:        [-2  -1   0   1   2   3]
              ↓   ↓   ↓   ↓   ↓   ↓
缩放后：    [-102 -51  0  51 102 153]
加 102：      [0  51 102 153 204 255]
              ↓   ↓   ↓   ↓   ↓   ↓
uint8:        [0  51 102 153 204 255]
            充分利用全部空间 ✓
```

**作用**：将权重的最小值映射到整数的最小值（0 或 -128），从而充分利用整数空间。

## 3. 量化的分类方式

### 3.1 按训练阶段分

| 方法 | 全名 | 说明 | 精度损失 | 计算成本 |
|------|------|------|--------|--------|
| **QAT** | Quantization Aware Training | 从头用低精度训练 | <1% | 高（完整训练） |
| **PTQ** | Post-Training Quantization | 训练后直接量化 | 1-5% | 低（无需重新训练） |

#### 理论效果对比

```
ResNet50 在 ImageNet 上：

float32 baseline:     76.13% Top-1
↓
PTQ (int8):           75.10% (损失 1.03%)
QAT (int8):           76.02% (损失 0.11%) ✓ 更优

极端量化（int4）：
PTQ:                  73.50% (损失 2.63%)
QAT:                  75.80% (损失 0.33%) ✓ 更优
```

#### 为什么 PTQ 更常用

1. **成本低**：不用重新训练，只需校准几百张图片
2. **便捷**：现成模型直接量化即可部署
3. **够用**：1-3% 的精度损失对大多数应用可接受
4. **成熟**：工具链完善（ONNX Runtime、TensorRT 等）

### 3.2 按精度等级分

```
float32 (32-bit)  标准精度
  ↓
float16 (16-bit)  半精度量化 ← 50% 显存
  ↓
int8 (8-bit)      常见工业级 ← 75% 显存
  ↓
int4 (4-bit)      LLM 主流   ← 87.5% 显存
  ↓
int2, int1        极限压缩（研究阶段）
```

### 3.3 按量化对象分

| 对象 | 说明 | 主要作用 |
|------|------|--------|
| **权重量化** | 只量化模型参数 | 节省显存 |
| **激活值量化** | 只量化中间计算结果 | 加速计算 |
| **混合量化** | 两者都量化 | 显存 + 速度都优化 |

### 3.4 按量化粒度分

```python
# 1. 逐层量化 (Per-layer)
#    整层用同一个量化参数
int8_weight = quantize(layer.weight, scale=s, zero_point=z)
精度：损失 3-5%

# 2. 逐通道量化 (Per-channel)
#    每个输出通道用不同的量化参数
for i in range(output_channels):
    int8_weight[:, i] = quantize(weight[:, i], scale=s_i, zero_point=z_i)
精度：损失 1-2% ✓

# 3. 逐组量化 (Per-group)
#    通常用于 LLM，每 128 个权重一组
#    LLM 中广泛使用的平衡方案
```

### 3.5 按对称性分

| 方式 | 范围 | 说明 |
|------|------|------|
| **对称量化** | [-a, a] → [-128, 127] | 简单，精度略低 |
| **非对称量化** | [min_val, max_val] → [0, 255] | 充分利用空间，精度更好 ✓ |

### 3.6 按动态/静态分

| 方式 | 说明 | 适用场景 |
|------|------|--------|
| **静态量化** | 量化参数固定（calibration 确定） | 推理、服务 |
| **动态量化** | 量化参数在运行时动态计算 | 推理，更灵活但较慢 |

## 4. 量化的完整流程

### 4.1 Calibration（校准）

```python
# 目的：在代表性数据上找到权重的真实分布范围

def calibrate(model, calibration_data):
    """
    在校准数据上运行模型，收集每层权重的统计信息
    """
    for layer in model.layers:
        min_val = float('inf')
        max_val = float('-inf')
        
        # 遍历校准数据，找最大/最小值
        for data in calibration_data:
            output = layer(data)
            min_val = min(min_val, output.min())
            max_val = max(max_val, output.max())
        
        # 计算量化参数
        layer.scale = (max_val - min_val) / 255
        layer.zero_point = -min_val / layer.scale
        
    return model
```

### 4.2 量化权重

```python
def quantize_weight(float_weight, scale, zero_point, bits=8):
    """
    将浮点权重量化为整数
    """
    max_int = (2 ** (bits - 1)) - 1  # int8: 127
    min_int = -(2 ** (bits - 1))     # int8: -128
    
    int_weight = round((float_weight - (-scale * zero_point)) / scale)
    int_weight = clip(int_weight, min_int, max_int)  # 防止溢出
    
    return int_weight
```

### 4.3 推理时反量化

```python
def forward_quantized(input, int_weight, scale, zero_point):
    """
    推理时用整数权重计算，然后反量化
    """
    # 用整数做计算（速度快）
    output = matmul(input, int_weight)
    
    # 反量化回浮点数
    output = (output - zero_point * input.sum()) * scale
    
    return output
```

### 4.4 完整的 PTQ 流程

```python
import torch

# 1. 加载模型
model = load_pretrained_model()

# 2. Calibration
calibration_loader = load_calibration_data()
model.eval()

for layer in model.layers:
    for data in calibration_loader:
        _ = model(data)  # 收集统计信息

# 3. 量化

# 4. 验证精度
accuracy = evaluate(quantized_model, test_data)
print(f"精度: {accuracy:.2%}")

# 5. 保存

```

## 5. 为什么量化有效？

### 5.1 神经网络对相对关系敏感

```
原始权重：      [0.1, 0.5, 0.9, 1.5]
简单截断：      [0, 0, 0, 1]      ← 完全丢失信息 ❌

量化映射：       [26, 57, 89, 154]
相对关系保持：   0.1:0.5:0.9:1.5 ≈ 26:57:89:154 ✓
```

### 5.2 具体例子

```
一个简单的线性层：

原始：y = w1*x1 + w2*x2
     其中 w1=0.1, w2=0.5

量化到 int8（scale=0.01, zero_point=50）：
     w1_int = 60, w2_int = 100

计算：y_int = 60*x1 + 100*x2

反量化：y = (y_int - zero_point) * scale
        = (60*x1 + 100*x2 - zero_point) * scale
        
虽然有误差，但 w1:w2 = 0.1:0.5 的比例保持了
↓
模型的决策逻辑保持不变
↓
精度损失 <2%
```

### 5.3 权重分布的自然特性

```
神经网络初始化（Xavier/He）：
权重 ~ Normal(0, std)
std = sqrt(2 / (fan_in + fan_out))

示例：1000→1000 的层
std ≈ sqrt(2/2000) ≈ 0.032

分布特性：
- 99% 的权重在 [-0.1, 0.1] 范围内
- 权重自然聚集在较小的范围
- 这正好适合整数表示！
```

## 7. 实际项目示例

### 7.1 快速量化 BERT

```python
from transformers import AutoModelForSequenceClassification
import torch

# 加载模型
model = AutoModelForSequenceClassification.from_pretrained("bert-base-uncased")

# 一行代码量化
quantized_model = torch.quantization.quantize_dynamic(
    model, 
    {torch.nn.Linear},  # 量化哪些层
    dtype=torch.qint8
)

# 验证
original_size = sum(p.numel() * 4 for p in model.parameters()) / 1e6  # MB
quantized_size = sum(p.numel() * 1 for p in quantized_model.parameters()) / 1e6

print(f"原始模型: {original_size:.1f} MB")
print(f"量化后: {quantized_size:.1f} MB")
print(f"压缩率: {(1 - quantized_size/original_size)*100:.1f}%")

# 结果：
# 原始模型: 440 MB
# 量化后: 110 MB
# 压缩率: 75%
```

### 7.2 LLM 快速量化

    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,

# 一行代码加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b",

## 9. 常见问题

### Q1: 量化会导致模型精度严重下降吗？

**A**: 不会。工业级量化（int8）通常精度损失 <2%，在大多数应用中可以接受。

```
例：ImageNet top-1 精度
float32: 76.13%
int8:    75.10%  (损失 1.03%)
int4:    75.80%  (损失 0.33% 用 QAT)
```

### Q2: 什么时候应该用 QAT 而不是 PTQ？

**A**: 当精度要求极高或你要用激进量化（int4、int2）时。PTQ 足以应付大多数场景。

```
PTQ：1-3% 损失（通常够用）
QAT：<1% 损失（确保最优）
```

### Q3: 量化后的模型还能继续微调吗？

**A**: 可以。这叫 Quantization Aware Fine-tuning，效果更好。

### Q4: 不同层应该用不同的量化参数吗？

**A**: 是的。逐层或逐通道量化能充分利用整数空间，精度更好。

## 10. 总结

### 量化的核心概念

```
量化 = 将浮点数的实际分布范围线性映射到整数范围

关键参数：
- scale: 缩放因子 = 范围 / 2^bits
- zero_point: 平移参数 = 把 min_val 映射到 0

结果：精度 <2% 损失，显存减少 75-87.5%
```

### 实施建议

```
1. 95% 场景有现成工具，不需要自己实现
2. 优先用框架自带量化（PyTorch、HF）
3. 推理部署用 ONNX Runtime 或 TensorRT
4. LLM 用 BitsAndBytes 或 llama.cpp
5. 性能不满足再考虑 QAT 或其他优化
```

### 学习路径

```
初级：理解量化的基本概念和公式
中级：掌握不同量化方式的选择

```
