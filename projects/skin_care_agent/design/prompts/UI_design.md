【背景】
我们正在开发一款痘痘/祛痘皮肤护理相关的产品。目前已完成 App 端第一次 UI 打样（MVP 阶段），形态基本满意，但距离正式上线的视觉预期仍有差距。产品的核心设计与功能你已了解，此处不再赘述。
产品定位补充：
目标用户：14–40 岁的年轻及中青年群体
因此配色需兼顾年轻活力与一定的专业可信感，避免过于低龄或过于医疗冷硬 。产品定位：你可从skin_care_agent中的project_background.md获取。
【本轮任务】
在正式设计视觉页面前，先完成一份市场配色调研（仅收集，暂不做分析）。调研市面上与"皮肤护理 / 祛痘 / 护肤记录"功能相关的成功产品，整理其主要配色方案。
【具体要求】
筛选标准：选取与我们产品定位契合、市场表现成功、拥有广泛用户群体的产品。按地区分组：欧美 / 日韩 / 中国，每个地区选 3–4 款，总计约 9–12 款，以便对比不同市场的审美差异。
每款产品需包含：
产品名称、所属公司、地区、简要定位
主色调 / 辅助色 / 强调色（Accent） 分别列出
每个颜色提供详细数值：HEX、RGB、HSB 或 HSL（含色相 H、饱和度 S、明度 B/L），如涉及透明度请标注
若该产品有多组配色方案（如浅色/深色模式、不同功能模块配色），请分别列出主要的几组
来源/配图：由于你可能无法直接提供真实截图，请为每款产品提供官网链接、App Store / Google Play 链接，以便我自行核实。若能用文字描述其界面视觉特征（整体风格、氛围）更佳。
本轮仅做调研收集，暂不需要分析和推荐结论。分析与配色定案放到下一轮。
输出格式：按地区分组，每组内用表格呈现，便于横向对比。
