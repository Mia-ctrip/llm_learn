## 📖 会话启动流程（必须执行）

在回答每个新会话的第一个问题前，必须先读取以下文件：

1. `project_background.md`
2. `backend/dev_notes.md`（唯一的开发进度记录文件）
3. `CLAUDE.md`

读取后先用一句话说明：当前主线步骤、已经完成的范围、当前未完成事项和下一步方向，然后再回答用户问题或开始开发。

不要把 `docs/dev_notes.md` 或 `docs/temp_step.md` 当作最新进度来源；它们是历史/补充文档。除非用户明确要求，否则不要在这两个文件中记录新的开发进度。

## 📝 开发进度日志规则（必须执行）

唯一日志文件是：

```text
D:\agent\model\projects\skin_care_agent\backend\dev_notes.md
```

开发期间不能只在用户说“结束”“再见”时记录。完成以下任一里程碑后，必须立即更新日志：

1. 完成一个完整 Step；
2. 完成一个可独立验证的子步骤，例如 3a、3b、3c；
3. 完成一个 API、数据库迁移、服务模块或核心算法；
4. 修复一个重要 bug；
5. 因等待用户、环境或外部条件而暂停；
6. 自主工作即将暂停、切换任务或可能发生上下文压缩前。

如果一次会话完成多个里程碑，每个里程碑都要分别记录，不要等全部工作结束后凭记忆补写。

每条记录按实现时间顺序追加，不能修改历史完成记录来掩盖后来变化。若同一子步骤有后续修正，追加“修正/补充”记录并引用原步骤。

每条记录至少包含：

```markdown
## YYYY-MM-DD — Step / 子步骤名称 — 状态

### 本次完成
- 完成了什么
- 新增或修改了哪些关键文件
- 完成到什么程度

### 验证情况
- 执行了哪些测试、迁移或接口验证
- 哪些通过，哪些尚未验证

### 当前阻塞或遗留
- 没有则明确写“无”

### 下一步
- 下一次应继续做什么
- 需要用户操作时明确写出操作内容
```

状态只能使用：

- `✅ 已完成`：实现和必要验证均完成
- `🚧 进行中`：已有实现，但仍有明确剩余工作或验证
- `⏸ 阻塞`：等待用户、环境或外部条件

记录前先检查日志末尾，避免重复记录；但不能因为已有旧记录就省略本次新增的实现、验证结果、遗留问题或下一步。

## 🔴 交互规则

1. 使用中文回复，简洁
2. 不要擅自安装依赖或执行破坏性命令
3. 修改代码后，按风险执行必要的测试或静态检查，并把结果写入进度日志
4. 如果无法验证，不得写成“已完成”，应写明“待验证”及原因
