# 《AI 驱动的痘痘管理产品》完整评估与执行报告

## 第一部分：核心定位（决定生死的一句话）

> **这是一个"AI 驱动的长期皮肤管理产品"，不是"AI 看病产品"。**

这个定位决定了后面所有产品设计、合规路径、商业模式。**定位错了，全盘皆错。**

**两个产品的本质区别：**

| 用户以为想要的 | 用户真正持续需要的 |
|----------------|--------------------|
| AI 帮我分析这颗痘是什么 | **有人持续关注我的皮肤** |
| 推荐我用什么药 | **知道我到底有没有在变好** |
| 一次性建议 | **告诉我下一步该怎么做、帮我坚持** |

→ 你做的是**右边那个产品**。左边是钩子，右边是护城河。

## 第二部分：最终结论（不灌鸡汤）

| 维度 | 判断 |
|------|------|
| 能不能做出来 | ✅ 概率 85%+，技术不是风险 |
| 需求是否真实 | ✅ 真实，人群大、痛点强、爱分享 |
| 能否产生种子用户 | ⚠️ 能（70%+），但规模化是最大不确定性 |
| 能否赚到钱 | ⚠️ 赚零花 40-50%，养活自己 15-20%，做成生意 5% |
| 最大风险 | ❌ 不是技术、不是合规，是**留存 + 获客** |

**最终建议：**
> **值得投入 1 个月做 MVP 验证，但绝不建议一开始就投入半年做完整版。**
> **做不做的真正决定因素：你愿不愿意、能不能搞定"获客"和"让用户持续记录"——而不是 AI 准不准。**

## 第三部分：四个阶段推进路线

### 阶段 0：先验证是不是伪需求（开发前必做，1-2 周）

**创业第一步不是写代码，是验证需求。**

**目标：访谈 50 个真实痘痘用户。**

**渠道：** 小红书、痘痘贴吧、豆瓣油痘肌小组、微信群、医美群。

**必问 8 个问题：**
1. 长痘多久了？
2. 最大的困扰是什么？
3. 看过医生吗？花了多少？
4. 买过哪些药/护肤品？
5. 最痛苦的环节是什么？
6. 是否愿意**每天**拍照记录？（关键，测留存意愿）
7. 是否愿意为"长期追踪 + 诱因分析"付费？
8. 能接受多少钱？

**通过标准：**
> 50 人里 **≥30 人**表示"如果真能长期追踪变化，我愿意尝试" → 才值得做。

---

### 阶段 1：MVP（4 周，只做最核心）

**MVP 目标不是赚钱，是验证"用户愿不愿意持续用"。**

#### 功能清单（只做 5 个）

**① 拍照记录（标准化引导）**
- 引导固定角度/光线/距离（否则 AI 分析一致性极差，这是真坑）
- 可做参考线框 + 上次照片半透明叠加，帮用户对齐

**② AI 状态分析（描述，不诊断）**
- 调用 GPT-4o / Gemini 2.5 Pro / Claude Vision，**第一版不训练模型**
- 输出统一 JSON，方便后续统计：
```json
{
  "closed_comedone": 12,
  "papule": 5,
  "pustule": 2,
  "severity": "medium",
  "skin_index": 72
}
```
- **话术红线：**
    - ❌ "你长的是脓疱型痤疮"
    - ✅ "图片显示下巴区域存在疑似脓疱样皮损约 2 处，整体炎症程度中等，建议结合专业医生意见判断"

**③ 趋势追踪（核心价值，用户感知最强）⭐**
- 日报 / 周报 / 月报
- 状态指数曲线
- 示例：
  > 第 1 天：炎症痘 28 个 → 第 30 天：炎症痘 14 个，改善 50%
- **这是用户"看得见自己变好"的地方，是付费理由的根。**

**④ 痘痘日记（多因素打卡）**
- 快捷标签：睡眠 / 经期 / 压力 / 辛辣 / 甜食 / 新产品
- 为后续诱因关联分析积累数据

**⑤ AI 问答（护理建议，非医疗）**
- 例："今天有红肿痘能化妆吗？" → 给护理/遮瑕建议
- 绝不给诊断和用药指导

#### MVP 阶段先不做：
复杂诱因算法、药物识别、产品档案、深度报告 —— **等留存验证后再投入。**

---

### 阶段 2：留存验证通过后再做（2-3 个月）

**① 药物/产品识别（拍药盒/扫码建档）**
- 识别阿达帕林、班赛、克林霉素、夫西地酸等 → 建立**档案**
- 提供**客观成分科普**：用途、注意事项、常见不良反应
- **绝不说：** 一天擦几次、擦多久、和哪个药联用 ←这是医疗红线
- **用户问"该用哪个"** → 给结构化成分对比表 + "建议咨询医生/药师决定"，把决策权交还用户

**② 经期 × 爆痘关联模型（女性强粘性功能）⭐**
- "过去 3 个月，你的状态指数下降通常出现在经期前 5 天左右"
- 用"相关性观察"措辞，不做医学因果断言

**③ 护肤品资产管理**
- 记录洁面/面霜/防晒使用历史
- 成分叠加冲突提醒（护肤常识，非用药）

---

### 阶段 3：真正的差异化护城河（有用户后才做）

**核心：Agent 长期记忆**

用户用满 90 天后，系统记住：经期规律、用药历史、爆痘位置、情绪、睡眠 → **主动提醒**：

> "你过去 3 个月经期前 4 天经常下巴爆痘，本周可提前加强护理。"

**这才是免费 GPT 做不到的、迁移成本极高的壁垒。**

## 第四部分：合规设计（产品的护身符）

### 永远不做的 4 件事
1. 不判断病种
2. 不推荐/指定药品
3. 不指导用药方式
4. 不替用户决定用哪个

### 话术替换表
| 禁用 | 替换 |
|------|------|
| 诊断/确诊/病因 | 状态记录/外观特征 |
| 这是 XX 痤疮 | 呈现"红肿凸起"疑似特征 |
| 推荐你用 X 药 | 含某成分的产品有这几个（科普） |
| 对症用药 | 区分这类情况需面诊，建议咨询医生 |
| 治疗/消炎 | 护理/舒缓 |

### 三道硬性机制
1. **首启强制免责声明**：明确"不提供诊断、不推荐药品、不替代诊疗"
2. **就医引导硬规则**：大面积/化脓/持续恶化 → 强制弹"建议就医"并停止护肤建议
3. **AI 输出层敏感词过滤**：防模型说漏嘴

## 第八部分：商业模式

**第一阶段不赚钱，先验证留存。**

| 模式 | 说明 |
|------|------|
| 会员订阅（核心） | 9.9-19.9 元/月：无限分析、档案、趋势报告、诱因分析 |
| 导购 CPS | 护肤品/防晒佣金，放内容区，勿污染分析区 |
| 医生导流 | 重度用户转线上问诊 |
| 品牌合作 | 流量起来后再做 |

**破解"成功即流失"悖论：**
- 把定位从"祛痘"扩展为**"终身皮肤管理"**：痘好了还有抗老、敏感肌、换季护理、产品资产
- 诱因档案是用户资产，越用越值钱 → 高迁移成本

## 第十部分：最务实的执行计划（关键！）

**最大的认知升级：不要先埋头开发半年，要同步验证获客。**

| 时间 | 产品线 | 获客线（同等重要） |
|------|--------|--------------------|
| 第 1-2 周 | 用户访谈 50 人 + 搭用户系统/拍照/AI 分析 | **开小红书账号，发"AI 记录我的祛痘曲线 + 遮瑕技巧"** |
| 第 3 周 | 趋势图 + 历史数据 | 持续发内容，观察能否起量 |
| 第 4 周 | AI 问答 + 日记，上线，拉 50 真实用户 | 看内容数据 |
| 上线后 30 天 | 观察留存指标 | 观察获客能力 |

**两个止损判断：**
- 若 **30 天打卡率 < 5%** → 重新思考定位，别加功能
- 若 **发遮瑕/记录内容也没人看** → 说明获客这关你过不了，**及时止损，沉没成本极小**
