1. 虽然还没有开始实践，但是我预计该产品能积累沉淀的用户数据可能是项目中最有价值的东西，比起订阅费/广告费，用户数据可能价值更大，所以如何完整妥善的存储用户数据是项目设计的重中之重；
2. 项目的数据除了基础的prompt文本,用户上传的图片也尤其重要；
3. 我任务项目是需要在用户上传图片上下功夫的，原因：
   a.用户对个人皮肤状态的描述和痘的定位无法仅通过语言描述清楚；
   b.图片能帮助该产品对所有用户更有针对性的给出诊断；
   c.该产品的定位是一个长期皮肤维护助理，因此非常依赖用户阶段性的上传皮肤状况来跟踪用户的痘痘发展和变换状态（痘痘瘪了？变成痘印了？红痘印/黑痘印？某个位置近期又冒新痘了？）
4. 本产品的idea来源于我每日和gpt交流沟通我的痘痘状况，对话式的ai app已经足够完成我产品最核心部分的功能，我需要产品对比ai app有明显的优势，我认为可以在以下几点上体现产品的特点：
   a. 重点放在图像识别和处理上，对比用户直接自拍+一段简单的描述发给llm, 我希望我的产品能在调用llm api前通过图像预处理来帮助llm更好的判断皮肤状态。例如：
      i. 自拍照片必须符合我产品制定的模板，类似人脸验证的核验引导框；
      ii. 主动对脸上非痘痘发作区域打码，尤其是眼睛；
      iii. 先通过图像识别，将脸上所有痘痘位置都圈点出来，标号&将本次图像和以往图像对比，根据标号判断出每个痘痘的生命周期的变换（冒痘，发炎，破皮，瘪痘，留痘印，痘印消失等）
   b. 重写promot，沉淀出一套能准确获取我产品需要返回的llm提问prompt，效果肯定比用户自己五花八门的描述更好；
   c. memory系统，除了上面提到的记录每颗痘痘的生命周期，我的产品还需要记录用户爆痘前的可疑饮食/不良习惯。（爆痘通常不是前一天饮食导致的，可能是一个星期前的饮食导致的，所以我产品记录下来很有意义。用户可能通常不会主动怀疑自己的饮食，坦白自己在饮食上的放纵，产品会以主动问询&点击回复的方式获得答案：睡眠 / 经期 / 压力 / 辛辣 / 甜食 / 新产品。）。产品还可以记录用户使用的外用类药品（**仅仅是记录**），引导用户自己发现各类产品对自己痘痘症状的效果（我作为被痘痘困扰多年的患者，使用过的外用药数不胜数，但到现在也不明确某种症状使用什么外用药是最有效的）。
5. 调用AI大模型帮助诊断和识别的part， 我认为一定需要是一个健壮的ai gate away， 仅使用一家厂商的一种模型肯定是不可以的，涉及到不同模型的路由，请求的分发，请求重试和模型降级；
6. llm api的选择 耗时优化 吞吐量优化在用户量上去后很重要，toC的产品看重infer阶段prefill的首token耗时， mvp阶段暂时不用考虑； 
7. 后续还可以考虑妆容建议（什么颜色的遮瑕/多用保湿产品软化角质/痘痘发炎部位不要厚遮粉底薄薄一层即可）但mvp验证阶段不需要该功能
8. 后续拓展还可考虑斑点/美容/整容恢复期记录
