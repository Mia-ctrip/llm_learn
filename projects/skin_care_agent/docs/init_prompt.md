## 一、产品目标（理解设计取向，不要超范围实现）

这是一个「AI 驱动的长期皮肤（痘痘）管理产品」，核心价值是**让用户看见自己皮肤变好的趋势**，而不是"AI 看病"。请牢记：

- **产品本质**：长期追踪 + 趋势可视化，不是一次性诊断工具
- **合规红线**：只描述外观状态，记录用户的皮肤/饮食/生活/用药等信息，**绝不诊断病种、不推荐药品、不指导用药**
- **MVP 目标**：验证用户「愿不愿意持续每天拍照记录」，不是赚钱

## 四、MVP 功能范围（只做这 5 个，多余的一律不做）

### 1. 拍照记录（标准化引导）
- 拍照页提供参考线框
- 显示上一张照片的**半透明叠加**，帮用户对齐角度
- 原图上传到 storageService
- AI 识别后，生成一张**标注图**（框出痘痘位置并编号）一并存入 storageService

### 2. AI 状态分析（描述，非诊断）
- 上传后调用 `aiService.analyze(imageUrl)`
- 返回统一 JSON 结构（下方为示例，字段可与我讨论后调整）：
    ```json
    {
      "closed_comedone": 12,
      "papule": 5,
      "pustule": 2,
      "severity": "medium",
      "skin_index": 72,
      "annotated_image_url": "...",
      "description": "图片显示下巴区域存在疑似脓疱样皮损约2处，整体炎症程度中等，建议结合专业医生意见判断"
    }
    ```
- ⚠️合规红线：`description` 必须是「描述外观 + 建议咨询医生」的合规话术，**绝不能出现诊断/推荐药品类表述**

### 3. 趋势追踪（核心功能 ⭐）
- skin_index 时间曲线
- 炎症痘数量随时间变化
- 支持「日报 / 周报 / 月报」切换
- 示例文案："第1天炎症痘28个 → 第30天14个，改善50%"

### 4. 痘痘日记（多因素打卡，与当天照片关联）
- **4.1 饮食生活**：睡眠 / 经期 / 压力 / 辛辣 / 甜食 / 新产品 等快捷标签
- **4.2 外用药记录**：阿达帕林 / BPO / 夫西地酸 / 鱼石脂软膏 / 莫匹罗星 / 红霉素 等
  - ⚠️合规红线：这里只是**用户主动记录"我用了什么"**，是用户行为日志，**系统不得给出"该不该用、怎么用、用多久"的建议**

### 5. AI 问答（护理建议，非医疗）
- 简单对话框，调用 `aiService.chat(question)`
- system prompt 强制约束：只给护理/遮瑕建议，**不诊断、不开药**

## 五、合规机制（必须内置）

1. 首次启动弹**强制免责声明**：「本产品不提供医疗诊断、不推荐药品、不替代专业诊疗」
2. `aiService` 内置**敏感词过滤层**，拦截"确诊 / 治疗 / 推荐用XX药 / 对症用药"等输出
3. 预留**就医引导**规则函数 `checkNeedDoctor(result)`：当 `severity === 'high'` 或痘痘数量超阈值时返回 true
4. 关键合规逻辑处，代码注释标明「⚠️合规红线」
