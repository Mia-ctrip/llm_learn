# Skin Care Agent — 项目背景

## 产品定位

AI 驱动的**长期皮肤管理伴侣**（iOS / Android App + FastAPI 后端），帮助痘痘用户通过每日拍照记录皮肤状态变化、发现生活习惯与爆痘的关联规律。

**核心价值**：不是 AI 诊断工具，而是长期追踪器 —— 让用户看到"第 1 天 28 颗 → 第 30 天 14 颗（改善 50%）"。

---

## MVP 五大功能（仅此五项）

1. **标准化拍照记录** — 参考框 + 叠加对齐，保证每次拍照角度一致
2. **AI 状态分析** — 返回 JSON（闭合粉刺/丘疹/脓疱数量、严重度、皮肤指数 0-100）
3. **趋势追踪** — 日/周/月报告 + 皮肤改善曲线
4. **痘痘日记** — 多因素打卡（睡眠、经期、压力、饮食、护肤品）
5. **AI 问答** — 护肤建议，非医疗诊断

---

## 核心技术差异点

- **区域生命周期追踪（Patch Tracking）**：以"痘斑区域"为单位追踪每个区域的密集度/主导痘型/炎症等级变化，覆盖从轻度零星痘到重度融合成片的全谱
- **单颗痘追踪（Point Tracking，轻度可选）**：`estimated_count < 10 且全为 sparse` 时激活，记住每颗痘的状态变化（新生 → 发炎 → 消退 → 留疤）
- **多照片横向比对**：当前照片与历史照片按 region 匹配 patch，趋势可视化
- **模式记忆**：爆发前 N 天的饮食/睡眠/压力数据，识别个人规律
- **图像预处理**：本地检查清晰度/光照/完整人脸/视角，并生成不调色的几何标准化副本

---

## 合规红线（硬性要求，不可妥协）

- 永远不诊断疾病类型（不说"你是细菌性痤疮"）
- 只描述外观状态（说"疑似约 2 颗含脓皮损，中度炎症"）
- 不推荐任何药品或用法
- 严重度超阈值时强制出现就医提示

## 技术栈

| 层级 | 选型 |
|------|------|
| 前端 | React Native + Expo SDK 57（iOS / Android） |
| 后端 | FastAPI + PostgreSQL + Alembic |
| 存储 | 本地文件系统（MVP） → 腾讯 COS（生产） |
| AI 视觉 | 多模型网关（Qwen-VL / GLM-4V / MiniMax / Doubao） |
| 图像处理 | MediaPipe + OpenCV + Pillow |
