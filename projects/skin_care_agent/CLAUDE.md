## 📖 会话启动流程（必须执行）

**在回答本会话的第一个问题前**，必须先执行：
1. 使用 Read 工具读取 `project_background.md`
2. 使用 Read 工具读取 `dev_notes.md`
3. 用一句话告知我项目当前阶段和开发进展
4. 然后再回答问题

---

## 📝 进度日志规则

每次会话结束前（我说"结束"、"再见"、"关闭"或明显收尾时），必须更新 `log_timeline.md`：

1. 追加一条新记录，格式如下：

```
## YYYY-MM-DD

**完成内容**：xxx

**下次继续**：xxx
```

2. 不要重复记录已有的内容，只追加新内容
3. 如果本次会话没有实质性进展，不需要记录
