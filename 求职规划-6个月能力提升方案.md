## 三、岗位决策

### 结论：主攻 AI Infra — 推理/Serving 方向，辅攻 AI 应用平台（Agent Infra/RAG Platform）

| 方向 | 匹配度 | 适配理由 |
|------|--------|----------|
| **AI Infra · 推理/Serving** | ⭐⭐⭐⭐⭐ 85% | ① GPU 调度经验直接平移 ② 推理层 Python/Java 均可，不被 Go 卡死 ③ 自学的 KV Cache/量化/vLLM 直接变现 ④ 96GB 可做有深度的 benchmark ⑤ 人才缺口最大 |
| **AI Infra · 训练平台** | ⭐⭐⭐⭐ 80% | 经验直接对口，但大厂训练平台团队以 Go 为主，面试会追问 |
| **AI 应用 · Agent Infra/Platform** | ⭐⭐⭐½ 70% | 进入门槛低、HC 多，但需要可展示项目产出来补短板 |

### 决策理由

**为什么推理/Serving 方向是最优解？**

1. **技能迁移成本最低**：GPU 资源调度 → 推理实例调度、配额管理 → 推理服务 SLA 管理、成本治理 → 推理成本优化，经验直接平移
2. **语言栈限制最小**：推理 Serving 层（vLLM 包装层、模型网关、路由层）Python/Java 均可，不像训练框架强依赖 Go/C++
3. **自学内容直接变现**：KV Cache、量化、Batching 策略是推理优化核心知识，已有数学级理解
4. **市场需求最旺**：2026-2027年推理成本是所有 AI 公司的 Top3 痛点，做推理优化的人供不应求
5. **有 GPU 可实操**：96GB 多卡可部署 72B 模型做真实 benchmark，产出可写进简历

### 项目 B：Code Review Agent（辅项目，体现 AI 应用能力）

#### 定位

展示 Agent 开发能力 + MCP 协议理解 + 实际应用落地能力。

#### 技术选型

```
语言：Python
框架：LangGraph（状态机编排）
协议：MCP（Model Context Protocol）
推理后端：项目A 的推理网关（自建）
版本控制：Git integration
```

#### 为什么选这个场景

1. 你做过 3 年代码开发，能评判 review 质量
2. 涉及 Tool Use / Structured Output / Multi-step reasoning
3. 可接入 MCP 协议（读 Git 仓库），展示对 Agent 标准协议的理解
4. 实用性强，面试能现场演示

#### 功能设计

```
输入：Git diff / PR URL / 本地代码变更
  │
  ▼
┌──────────────────────────────────┐
│ Step 1: Diff 解析                │
│ - 按文件分片                     │
│ - 提取变更上下文                 │
└──────────────┬───────────────────┘
               │
  ▼
┌──────────────────────────────────┐
│ Step 2: 并行 Review              │
│ - 每个文件独立调用 LLM           │
│ - Structured Output schema      │
└──────────────┬───────────────────┘
               │
  ▼
┌──────────────────────────────────┐
│ Step 3: 聚合 & 去重              │
│ - 合并相似 findings              │
│ - severity 排序                  │
└──────────────┬───────────────────┘
               │
  ▼
输出：JSON 格式 review report
  - severity (critical/warning/info)
  - category (bug/perf/style/security)
  - suggestion (具体修改建议)
  - code_snippet (相关代码片段)
```

#### MCP 集成

提供 MCP Server 工具能力：
- `git_read_file` — 读取仓库中指定文件
- `git_get_diff` — 获取指定 commit/branch 的 diff
- `git_list_commits` — 列出最近 commit 历史
- `git_get_context` — 获取变更文件的周围上下文

#### 里程碑

| 时间 | 产出 |
|------|------|
| 9月中 | MVP：单文件 diff → LLM review → JSON 输出 |
| 9月底 | 多文件并行 + MCP Server 实现 |
| 10月 | 去重/聚合逻辑 + CLI 工具化 |
| 11月 | README + demo 视频 + 博客 |

#### 简历效果（预期）

> 基于 LangGraph 构建多步骤代码审查 Agent，集成 MCP 协议实现 Git 仓库工具能力，支持 diff 分片并行 review + 结构化输出（severity/category/suggestion），使用自建 72B 推理后端。
